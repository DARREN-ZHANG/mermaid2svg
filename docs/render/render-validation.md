## 8. Determinism and safety notes

- The renderer uses `securityLevel: "strict"` and `suppressErrorRendering: true`.
- Each render call uses a unique id (`"m2s-" + ++counter`) to avoid id
  collisions across the 18 in-page renders.
- The test harness loads the renderer module into a real Chromium page via Vite
  middleware (so `mermaid` and its dependency graph resolve correctly), then
  calls `window.__m2s.renderMermaidToSvg(text)` per case.
- The capabilities report is written once in the `after()` hook after all
  renders complete, so its numbers are guaranteed to match the asserted results.
